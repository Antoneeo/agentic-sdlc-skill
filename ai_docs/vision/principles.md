# Vision Principles

## Principles
- **Vision before solution**: clarify intended benefit and non-goals before choosing implementation details.
- **Alignment over local optimization**: a technically correct change is not acceptable if it moves the product away from its declared direction.
- **Explicit conflict handling**: when a request conflicts with Vision, the agent must surface the conflict instead of silently implementing.
- **Lightweight governance**: Vision documents should be concise enough to read at the start of every feature.

## Strategic Anti-Patterns
- Treating feature history as a substitute for Vision.
- Writing technical analysis without naming the expected product benefit.
- Expanding scope because implementation is easy rather than because the Vision justifies it.
